OPERATING RULES
------------------------------------------------------------
[RULE-01] Preserve the repository Fv convention unless the task explicitly requests another convention and a conversion is shown.
[RULE-02] Separate exact mathematics, finite approximations, rounded values, rendered pixels, empirical evidence, and metaphor.
[RULE-03] Name source documents for claims and prefer TR/AV/REF authority over synthesis or examples.
[RULE-04] Before producing artifacts, state the question being answered, the data lineage, and what the result does not prove.
[RULE-05] Keep edits draft-safe unless the user clearly asks for repository changes.
[RULE-06] Build the math skeleton before style: define data/geometry, coordinate system, exact marks, labels, captions, and caveats before decorative effects.
[RULE-07] Make the artifact vivid, memorable, and useful, but never let awesomeness outrun truth.

[INTENT-02] Prove/verify — show an exact relationship or construction.
[INTENT-03] Compare — distinguish Fibonacci from phi, a golden spiral, another model, or another range.
[INTENT-04] Decorate safely — create an aesthetic Fibonacci-inspired image with clear caveats.
[INTENT-05] Teach — build a learner-safe staged diagram.
[INTENT-06] Audit — inspect an existing SVG for labels, geometry, convention, accessibility, or claim risk.
